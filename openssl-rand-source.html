How does OpenSSL Generates Keys?

Static Analysis

Web Search

So one wonders what exactly does OpenSSL do to generate a random key.

The first thing I did, is a web search, giving answers such as this one which implies that OpenSSL uses internal CSPRNG, a Cryptographic Secure Pseudorandom Number Generator seeded by the OS CSPRNG, which is seeded from real entropy sources. Specifically, in modern Linux machines, it is seeded by the RDSEED instruction, which is a TRNG, true random number generator.

This might be true in general. But is this the case for my particular system, with its configuration and OpenSSL version? I can't be certain

OpenSSL Code

In the code, one can see that RAND_bytes calls RAND_byte_ex and then things become less clear. An internal "engine" of OpenSSL is selected to generate random bytes

We can search further in the code and see that the only production implementations of RAND_METHOD are crypto/engine/eng_rdrand.c and engines/e_padlock.c.

Seeds of Doubt

There seem to be two engines which can be used by RAND_bytes. How can we see which one is actually used in my machine? How can I be certain what OpenSSL actually does?

We wish to advance until after the call to RAND_get_rand_method. Summary

We minimized our test case, used ltrace to find library function calls, used x/i $pc to disassemble current location and adv to "move" the instruction pointer to a certain location. Finally the gdb commands catch syscall open and bt showed us who opens /dev/urandom.
